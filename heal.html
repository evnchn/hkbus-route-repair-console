removed CSDI defect patch (OSM-matched)
What this is. CSDI (the government dataset hkbus already uses) stays the baseline for every route. This page shows the only places a patch is applied: inter-stop segments where CSDI provably drew a defect — an out-and-back retrace (loop, self-overlap collapse ≥ 20%, the discriminator validated 26/26 vs KMB with 0 false positives) or a grossly inflated wrong circuit (circuit, ≥3× the road alternative and ≥800 m excess). The patch is the OSM map-matched path for that segment only, and is applied only if it passes road/turn QC itself; where the operator's own line (KMB/LWB search API) exists, a patch that disagrees with it is vetoed. Everything not shown here is byte-identical CSDI — routes whose geometry can't be decomposed safely are left untouched by design.

Maintenance profile: one-off. The patch set is a static artifact; no recurring human effort. Each patch is keyed to the defect it fixes, so if a future CSDI release fixes the defect upstream, the patch retires itself.
